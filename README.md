# ACP (Agent Communication Protocol)

让你的 AI Agent 能和其他 Agent 高质量协作，而你只负责复制转发。

ACP 是一套面向 **人类中转场景** 的 Agent 通信规范：消息结构化、可审阅、可追溯，且能跨 Claude Code / Codex / OpenClaw 使用。

## 为什么值得用

如果你已经在同时使用多个 Agent（本地开发、云端部署、测试验证、文档整理），协作效率往往不是卡在“能力不够”，而是卡在“沟通不可控”：

- 信息在人类转述中不断丢失，尤其是约束条件和上下文边界
- 同一个背景要反复讲，Agent 每次像“第一次合作”
- 任务交接缺少统一格式，容易出现理解偏差和返工
- 协作过程没有结构化留痕，复盘时很难定位问题

ACP 的价值不是“再加一个模板”，而是给多 Agent 协作补上一层可复用的通信基础设施：

### 1) 降低沟通损耗
- **人能审**：消息是 Markdown，可先审阅再转发
- **AI 能解**：固定字段（Type / From / Context / Ask）让对方快速抓重点
- **上下文可裁剪**：只传完成任务必需的信息，减少噪音干扰

结果是：同一轮沟通更短、误解更少、补充问答次数更可控。

### 2) 把“临时协作”变成“稳定协作”
- 通过 `agent-profile` 明确能力、边界、偏好
- 通过 `contacts/` 累积联系人档案和历史交互摘要
- 新 Agent 首次沟通先 Introduce，后续按档案对齐语境

结果是：你和同一个 Agent 越合作越顺，而不是每次重新磨合。

### 3) 让跨平台协作有统一语言
- 同一套协议可覆盖 Claude Code / Codex / OpenClaw
- 不同 Agent 工具链可以分工，但交接格式保持一致

结果是：平台切换成本下降，团队不需要为每个平台重写一套沟通方法。

### 4) 支持人机团队协同与管理
- 你可以和同事/伙伴的 Agent 做结构化进度同步
- 每条消息可回看，可用于复盘、交接、培训新成员
- 敏感信息默认脱敏，降低误传风险

结果是：协作过程可追溯、可交接、可规模化，而不是“只靠当事人记忆”。

### 5) 让今天就能落地
ACP 默认假设“人类中转”是现实常态，不要求复杂直连基础设施。你不需要改造现有系统，只要按协议发消息，就能立刻提升多 Agent 协作质量。


## 典型场景

1. **本地开发 + 云端部署**
   - 本地 Agent 写代码
   - 云端 Agent 部署运维
   - 人类做消息中转

2. **多 Agent 分工流水线**
   - A 做方案，B 实现，C 验证
   - 通过 ACP 串成一个稳定闭环

3. **跨平台协作**
   - Claude Code、Codex、OpenClaw 混用
   - 统一通信协议，降低切换成本

4. **需要留痕和复盘**
   - 每条消息可回看
   - 适合团队协作和长期迭代

## ACP 的核心设计

- **消息即教程**：首次通信可附协议摘要，不懂 ACP 也能快速上手
- **AI 自动选类型**：无需人手动选择 Ask/Reply/Briefing
- **首次建联先 Introduce**：先交换名片，再进入正事
- **敏感信息默认脱敏**：避免把 key/密码等内容带出

## FAQ

### Q1: 为什么不直接让两个 Agent 自己连？
A: 很多真实场景下，跨环境直连并不稳定或不可用。ACP 假设“人类中转”是常态，优先解决今天就能落地的问题。

### Q2: 如果对方 Agent 不懂 ACP 怎么办？
A: 首次消息可自带简版协议说明，对方照格式回即可。

### Q3: ACP 只是一个消息模板吗？
A: 不止。它的价值在于“结构化 + 上下文裁剪 + 记忆沉淀”这整套协作机制。
