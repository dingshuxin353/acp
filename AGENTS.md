# ACP 使用规范

本文件夹包含 Agent Communication Protocol (ACP) 的全部运行时文件。当需要与其他 Agent 沟通时，读取此文件。

## 文件说明

| 文件 | 用途 |
|------|------|
| `protocol.md` | 协议规范：消息格式、8 种类型定义、裁剪规则 |
| `agent-profile.md` | 本 Agent 的名片：身份、能力、边界 |
| `contacts/` | 通讯录：每个认识的 Agent 一个文件，含档案和对话记忆 |

## 何时使用 ACP

- 用户说"帮我跟 xxx 沟通/说/发消息" → 进入发送流程
- 用户粘贴了包含 `--- ACP START ---` 的内容 → 进入接收流程
- 用户用 `$agent-msg` 触发

## 发送流程

1. 读取 `protocol.md` 了解消息格式
2. 读取 `agent-profile.md` 了解自己是谁
3. 检查 `contacts/` 判断对方是否是新 Agent
   - **新 Agent** → 先生成 Introduce 消息交换名片，不直接进入正事
   - **已认识** → 读取对方档案，直接生成对应类型消息
4. 根据用户意图自动选择消息类型（不让用户选）
5. 从项目上下文裁剪相关信息
6. 生成消息，展示给用户审阅后复制转发

## 接收流程

1. 解析收到的 ACP 消息
2. 向用户简要说明"对方在说什么、需要你做什么"
3. 更新通讯录：
   - 新 Agent → 在 `contacts/` 创建新文件
   - 已有 Agent → 追加本次对话摘要
4. 判断行动（执行任务 / 回答问题 / 接收信息 / 记录 Remember）
5. 生成回复消息，展示给用户审阅后复制转发

## 通讯录管理

每个 Agent 一个文件，存在 `contacts/` 下。文件名用简短标识（如 `cloud-ops.md`、`design-agent.md`）。

每次对话结束后，在对应文件的"对话记录"部分追加本次摘要。

联系人文件格式：

```markdown
# [Agent 名称]

## 档案

- 首次接触：YYYY-MM-DD
- 能力：[从对话中提取]
- 边界：[对方不能做什么]
- 沟通偏好：[语言、格式等]
- 项目背景：[对方在做什么]
- 备注：[有用信息]
- 最近接触：YYYY-MM-DD

## 对话记录

### YYYY-MM-DD — [话题简述]
- 消息类型：Ask / Reply / ...
- 摘要：[一两句话概括这次对话内容和结果]
- 收获：[对方提到的有价值信息，比如技术细节、偏好、已知问题]

### YYYY-MM-DD — [话题简述]
...
```

## 核心规则

1. **消息必须经用户审阅** — 生成后展示，等用户确认再提示复制转发，绝不自动发送
2. **首次沟通先打招呼** — 新 Agent 必须先 Introduce 交换名片
3. **上下文宁少勿多** — 对方会追问，信息过载比不足更糟
4. **敏感信息脱敏** — API key、密码、内部路径不出现在消息中
5. **Remember 要落地** — 收到 Feedback 中的 Remember 字段，建议用户写入教训库
6. **每次对话后更新通讯录** — 新信息追加到联系人档案，对话摘要追加到记录
