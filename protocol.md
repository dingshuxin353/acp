# Agent Communication Protocol (ACP) v0.1

一种基于自然语言的 Agent 间异步通信格式。人类负责在 Agent 之间传递消息。

## 设计原则

1. **消息即教程** — 每条消息自带格式说明，对方 agent 不需要提前了解协议
2. **人能读** — 人是信使，消息必须人也能看懂和审阅
3. **AI 自判断** — 消息类型由 AI 根据场景自动选择，不需要人指定
4. **上下文裁剪是核心价值** — 不是格式本身，而是 agent 帮你把信息整理好

## 消息基本结构

所有消息共享同一个外壳：

```
--- ACP START ---
> [首次通信时的协议摘要，后续省略]

## Type
[消息类型]

## From
[发送方身份，1-3句]

[...类型特有字段...]

--- ACP END ---
```

### 首次通信的协议摘要

当对方 agent 可能不了解 ACP 时，在消息开头附带：

```
> 这是一条 ACP (Agent Communication Protocol) 消息。
> 如果你了解 ACP，直接按协议回复。
> 如果不了解，规则很简单：保持同样的段落结构回复即可。
> 必填：Type（消息类型）、From（你是谁）。其余字段按需使用。
```

后续通信省略此摘要。

## 消息类型

### Introduce — 能力声明

首次建联时交换身份信息。让对方知道你是谁、能做什么、怎么跟你协作最高效。

字段：
- **Capabilities** — 能做什么（列表）
- **Boundaries** — 不能/不会做什么（列表）
- **Preferred Style** — 偏好的沟通方式（可选）

### Ask — 发起任务

请求对方做一件事。

字段：
- **Ask** — 核心诉求
- **Context** — 对方完成任务所需的背景信息（裁剪后）
- **Expect** — 期望的回复内容和格式

### Reply — 任务回复

任务完成或有结论时的回复。

字段：
- **Status** — done / in-progress / blocked
- **Done** — 已完成的事项（编号列表）
- **Output** — 交付物（链接、文件、配置等）
- **Question** — 需要对方确认的问题（可选）
- **Suggest** — 执行过程中发现的改进建议（可选）

### Checkpoint — 过程同步

长任务执行过程中的阶段性汇报。

字段：
- **Thread** — 任务线程 ID
- **Step** — 当前步骤 / 总步骤
- **Done So Far** — 已完成的步骤
- **Next** — 下一步计划
- **Risk** — 风险或需确认的问题（可选）
- **No Action Needed If** — 什么条件下不需要回复，agent 自行继续（可选）

### Briefing — 项目交接

把整个项目的上下文移交给另一个 agent，让它能独立工作。

字段：
- **Purpose** — 为什么要交接
- **Project** — 项目基本信息
- **Architecture** — 技术架构或结构说明
- **Known Issues** — 已知的坑
- **Key Decisions Made** — 已做出的重要决策
- **Ongoing Concerns** — 持续关注的问题

### Share — 知识共享

分享一个发现、经验或有用的信息，不需要对方执行任何任务。

字段：
- **Topic** — 主题
- **Content** — 具体内容
- **Relevance** — 为什么跟对方有关

### Feedback — 反馈闭环

对之前任务执行质量的反馈。

字段：
- **Thread** — 关联的任务线程
- **Good** — 做得好的地方
- **Improve** — 可以改进的地方
- **Remember** — 希望对方以后记住的规则（对方 agent 应将此写入自己的教训/记忆）

### Relay — 多方转发

需要第三个 agent 介入时，打包之前的背景转发。

字段：
- **Original Thread** — 原始任务线程
- **Forwarding To** — 转发给谁
- **Background Summary** — 之前对话的压缩摘要
- **Ask** — 需要第三方做什么
- **Context From Previous Thread** — 从之前对话中提取的相关上下文

## 对话线程

多轮通信时使用 Thread 字段追踪话题：

```
## Thread
deploy-ai-course-20260306
```

规则：
- 首次通信由发送方生成（简短描述-日期格式）
- 后续消息沿用同一 Thread
- 对话轮次多时，在 Context 中压缩总结之前的全部对话，不要求人类保留历史

## 上下文裁剪规则

裁剪是 ACP 的核心价值。遵循以下原则：

1. **只给相关的** — 对方完成任务需要什么就给什么
2. **给结论不给过程** — "node >= 18" 而不是 "我们试过16不行后来升到18"
3. **已知的坑主动说** — 这是 agent 帮人沟通的核心价值
4. **敏感信息脱敏** — API key 用占位符，密码不传
5. **宁少勿多** — 信息过载比信息不足更糟，对方会主动追问

## 学习闭环

Feedback 消息中的 Remember 字段构成跨 agent 学习机制：
- 收到 Remember 的 agent 应将其沉淀到自己的教训库或 CLAUDE.md
- 这意味着 agent 之间可以互相训练，协作质量随时间提升
